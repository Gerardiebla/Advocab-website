User Guide

A vocabulary mining engine for serious language learners (C1+). Import text from multiple sources, filter out words you already know, approve the keepers, polish them with AI enrichment, and export to Anki as finished cloze cards — all locally, all on your machine.

Setup Workflow Batch Page LLM Features Shortcuts Troubleshooting

1. First-time setup

Installation & setup

  1. Download — grab RosellaDeckBuilder-lite.zip or the Inno Setup installer from the website.
  2. Install — extract the zip, or run the installer. Everything you need is bundled.
  3. Run the app — launch Rosella from the Start menu or its desktop shortcut.
  4. Setup wizard — on first run, it walks you through:
    • Selecting your learning language.
    • Choosing to install AI extras (spaCy, models, Ollama integration) on demand.
    • Pointing to your local Ollama server or remote LLM endpoint.
    • Verifying Anki + AnkiConnect are reachable.

Configuration: Settings persist in config.json beside the exe (you can edit it for advanced tweaks without restarting).

System requirements

Component What you need
OS Windows 10 / 11
LLM Ollama (free, local) or any OpenAI-compatible endpoint
Anki Anki Desktop + AnkiConnect add-on (open when minting/syncing)
Internet Optional (needed for remote LLM, TTS, image generation)

2. The workflow

Five-stage pipeline that turns raw text into Anki cards:

  1. Import — choose a source (Kindle, paste, URL, YouTube, CSV). The app extracts candidate words/phrases.
  2. Batch — review candidates in a fast, keyboard-driven table. Approve the ones worth learning, discard the rest.
  3. Queue — watch the background LLM worker mine idioms and process your batch (non-blocking).
  4. Finalise — polish each card (fix translation, add mnemonic, generate example, ask the LLM, add images/audio).
  5. Mint — push approved cards to Anki.
✓ Non-blocking background work: A background LLM worker mines idioms while you triage. An image scheduler can generate mnemonic images automatically.

3. Pages & features

Page 0: Import

Pick a source and choose how to process it:

Sources:

  • Paste / URL — paste text directly, or give a URL and click Fetch to pull article text.
  • YouTube — paste a video URL/ID; captions are downloaded and split by sentence.
  • Kindle — import words you looked up on a connected Kindle device (lemmatised, frequency filter bypassed).
  • CSV — upload a .csv or .tsv file; map which column is the term.

Ingest modes:

  • Single words — deterministic: spaCy extracts content words, filters by frequency, stages them for review. No LLM, instant results.
  • Phrases — the background LLM worker mines idioms and set phrases from each sentence asynchronously.
  • Both — run both tracks simultaneously.

Processing rules: Import as (Lemma/Surface/Both), keep POS, min length, frequency band, strip endings, exclude names/places, skip already-known, keep context, deduplication, harvest grammatical phrases.

Page 1: Batch

A dense, keyboard-first table of candidate words/phrases. Each row shows target, POS, gloss, frequency rank, register, context, and any custom columns.

Filtering: Frequency band, POS, register, names/places, unranked only, custom columns.

Keyboard shortcuts:

  • Enter — approve selected row(s) and auto-advance
  • Space — mark selected row(s) as known
  • Ctrl+A — select all visible rows
  • Ctrl+Enter — approve all selected rows
  • Ctrl+Z — undo last knockout

Approved words go to Finalise; phrases are also enriched and sent there.

Custom columns: Define your own LLM columns (e.g., "Classify into a category"). Click Add column, set label, type, and LLM prompt. Results render as a new column and are filterable.

Page 2: Queue

The control room for background processing:

  • LLM Queue — live list of running/pending jobs (phrase mining, word enrichment, custom column generation).
  • Worker controls — Start/Stop/Force Stop buttons.
  • Connection status — LLM light green when Ollama is reachable; amber if a model isn't pulled.
  • Settings tabs — per-function LLM configuration (Enrichment, Custom Columns, Ask the LLM, Image Generation, TTS).

Page 3: Finalise

The per-card polish bench, split into three panes:

Pending list (left): Mint selected (as-is), Enrich selected, Discard, or Select all.

Edit pane (right, top): Target (with ▶ to speak + IPA), compound breakdown, translation (with sense picker), grammar, gender/plural, notes, cloze example, mnemonic (text + images).

Buttons: Enrich selected (LLM polish), Generate images, Save (Ctrl+S), Mint.

Ask the LLM pane (right, bottom): Chat assistant. Inject card context (toggle), speak responses (toggle), stream live.

Page 4: My Vocab

Browse your entire vocabulary matrix. Columns: Status (click to toggle), Lemma, Frequency rank, Register, Source, SRS data (if synced).

Actions: Sync from Anki (pull SRS stats), Seed from Lute, Delete.

Page 5: Settings

Tabs: General, Language, LLM, Frequency, Grammar tags, Anki, Image Generation, TTS, Dictionary building, Advanced.

Page 6: Help

Quick reference — keyboard shortcuts, connection status, links to full docs.

Page 7: Lab

Experimental features:

  • Language Brain — 2D semantic graph of known/learning vocabulary. Nodes coloured by mastery (if SRS synced). Domain overlays, music modes.
  • Domains — track progress within semantic domains (Food, Travel, Work, etc.). Top gaps for each domain. Create custom domains from seed phrases.

4. LLM features (Intelligence)

All optional — the app works great for deterministic workflows (dictionary, lemmatization, frequency, IPA).

Core workflows

  • Enrich — picks best sense, generates mnemonic, creates fresh example, adds usage notes.
  • Custom LLM columns — define domain-specific metadata; generated as background jobs.
  • Ask the LLM — interactive chat with streaming output and optional card context.
  • Phrase mining — background worker continuously mines idioms from staged sentences.

Reasoning models

LLMs with chain-of-thought (thinking tags) fully supported. Extra token headroom automatically allocated. Enable in Settings → LLM → [function tab] → Reasoning model (toggle).

Master LLM switch

Global "Enable LLM processing" toggle in Settings → LLM. When unchecked: all LLM features disabled, deterministic features keep working.

Remote APIs & per-function routing

Route individual functions to OpenAI, Groq, Together, OpenRouter, etc. for cost savings or better models.

Setup: Settings → LLM → Remote APIs & function routing → Manage API profiles. Add a profile (name, base URL, API key, model), then assign each function (Import/Enrich/Custom Columns/Ask) to a profile.

5. Image generation (mnemonics)

Three providers:

  • Local diffusion (Stable Diffusion, private) — ~20–40 sec/image on modern GPU.
  • Pollinations (cloud, free) — excellent quality, ~5 sec/image. Requires internet.
  • Wikimedia embeddings (search Commons, free) — instant, depends on available images. Requires internet.

Scheduling: Immediate (on-demand) or Scheduled (daily batch at a set time).

Choose in Settings → Image Generation → Provider and Mode.

6. Audio (TTS)

Usage: ▶ in Finalise's Edit pane, batch TTS button, speak chat responses, auto-exported to Anki.

Four engines:

  • gTTS (default, online, free) — Google's TTS, auto-detected language. Fallback engine.
  • Local (pyttsx3) (offline, OS voices) — system voices, set voice and speech rate.
  • Remote (OpenAI-compatible) (any TTS API) — commercial voices (OpenAI, ElevenLabs, etc.).
  • Custom (Coqui XTTS) (advanced) — fine-tuned XTTS model locally (most users prefer Remote).

Choose in Settings → TTS or LLM Status page's TTS tab.

Fallback: If synthesis fails, silently falls back to gTTS. Check app log for details.

7. Anki integration

Minting cards

Mint (Finalise page) creates finished cards in your target Anki deck. Creates or grows the note type as needed. Marks the vocab entry as learning in the app (won't be re-mined).

Seeding from Anki

Seed from decks (Settings → Anki) marks words already in your Anki deck(s) as learning. Mining skips them (no duplicates).

SRS data sync

Sync from Anki (My Vocab page) pulls scheduling data from your Anki collection: review counts, ease factor, intervals, lapses, mastery score, maturity classification.

Use for: seeing progress in My Vocab (Maturity, Mastery %, Lapses columns), colouring Language Brain nodes by mastery, tracking progress within semantic domains.

Requirements: Keep Anki + AnkiConnect running when minting, seeding, or syncing SRS data.

8. Register (formality levels)

Register tags indicate formality (German only, for now):

  • Gehoben (elevated, formal, literary) — speeches, academic writing.
  • Standard (neutral, default) — everyday usage.
  • Umgangssprachlich (colloquial, informal) — casual speech, friends.
  • Jugendsprache (youth slang) — current slang, often ephemeral.
  • Vulgär (vulgar, offensive) — profanity; marked for awareness.

In the app: Shows on Batch rows (colour-coded). Filterable. Exported to Anki on minted cards. Used in domain-mastery gaps.

Certainty threshold: Settings → Register certainty (0–100; higher = stricter).

9. Multi-language support

10 languages: German, French, Italian, Spanish, Portuguese, Dutch, Russian, Japanese, Chinese, Greek. Japanese support is experimental.

Language switching

  1. Settings → Language → Active language (pick from dropdown).
  2. App downloads missing assets: spaCy model, frequency list, dictionary.
  3. Readiness indicator shows ✓ (ready) or ○ (missing) for each language.

Per-language data

Each language gets its own SQLite database (advocab_de.db, etc.) storing your vocabulary, phrases, SRS sync data, and domain definitions. Switching languages switches database automatically.

Per-language grammar tags

Settings → Grammar tags shows language-specific toggles (German: tense/mood/number/case/gender; Japanese: particle role/verb form/formality; others: fallback set).

Japanese (experimental)

  • Sudachi tokenizer — superior to MeCab for modern Japanese.
  • Furigana — readings with ruby annotations (enabled by default).
  • Particle detection — は/が/を/に/で/etc. recognized and prefixed with functional labels.
  • Dictionary ranking — POS-aware lookup (particles shown before syllables).
  • Spaceless text handling — no spaces; cloze and deduplication work correctly.

Gaps: Full JMdict dictionary (pending licence clarity), IPA disabled for JA (espeak-ng poor support).

11. Tips & troubleshooting

Setup & connectivity

  • "Anki not reachable" — make sure Anki is running and AnkiConnect is installed. Click Settings → Check connections.
  • "LLM unavailable" — the app offers to pull a missing model, or run ollama pull gemma4:e2b manually.
  • "Kindle light amber" — Kindle sync requires a connected device (USB) with a vocab.db file.

Dictionary & lemmatization

  • "Word has no definition" — likely a tricky compound or inflected form. Try Settings → Dictionary → Build dictionary.
  • "Odd frequency rank" — lemmatization edge cases. Try Settings → Frequency → Build lemma-grouped frequency.
  • "IPA and compound badges are blank" — optional extras not installed. Install in Settings → AI Extras.

LLM & background processing

  • "LLM queue keeps growing" — worker is slower than you're importing. Pause it (Queue → Stop) or give it more time.
  • "Worker crashes" — worker detects a crash, backs off, and after repeated crashes quarantines the batch rather than looping forever.
  • LLM processing disabled — if you toggled Settings → LLM → Enable LLM processing to off, re-enable to resume.

Image generation & audio

  • "Image generation failed silently" — check app log (advocab.image logger). Images fail gracefully.
  • "TTS voice sounds wrong" — you're using gTTS (default). Switch to Local (pyttsx3) for OS voices, or Remote for commercial voices.
  • "TTS fell back to gTTS" — your chosen TTS engine failed; check app log for why.

Anki export

  • "Cards didn't mint" — make sure Anki + AnkiConnect are running. Click Settings → Check connections.
  • "Note type mismatch" — the app will grow the existing note type (add new fields). Check your Anki deck to confirm.

Performance & disk space

  • "App is slow" — large imports or slow LLM inference are expected. Processing happens in background while you triage.
  • "Disk space growing fast" — images stored in web_media/ (~1–5 MB per 100 images). Audio stored in Anki's media folder. Both expected to grow.
